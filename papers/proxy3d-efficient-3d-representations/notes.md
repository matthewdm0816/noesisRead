## 费曼讲解

想象你走进一个房间，你的眼睛（视觉）接收到了大量的 2D 画面信息。你的大脑并不是把每一帧画面的每一个像素都记下来——那样信息量太大了，而且很多是冗余的。相反，你会自然地把房间里"同类的东西"归在一起：那边是一组椅子，那边是一个书架上的书，那边是桌上的杯子。你记住的是"有哪些东西"以及"它们在哪里"，而不是成千上万个像素点。

这就是 Proxy3D 的核心思路。

现有的视觉-语言模型（VLM）处理 3D 场景时，通常面临两个问题：

**第一类方法（correspondence-based）**：把多帧 2D 图像喂进去，通过帧与帧之间的特征匹配来"隐式"理解 3D。这就像你试图通过比较多张照片的相似区域来理解房间的空间结构——理论上可以，但实际上很容易出错（空间不一致性），而且需要很多计算。

**第二类方法（representation-based）**：先把 3D 场景显式建模（比如点云），再序列化成 token 送给语言模型。这更像是你先用激光扫描仪扫描房间，然后把扫描结果告诉别人。但问题是，扫描结果太长了——一个普通房间的点云可能有几万个点，如果每个点都是一个 token，语言模型要处理的序列就太长了。

Proxy3D 的解决方案非常优雅：**用语义聚类来压缩 3D 表示**。

具体来说：
1. 从视频帧中提取 2D 语义特征（用预训练的视觉编码器）和 3D 几何信息（用点图预测器）
2. 利用 2D 分割模型（SAM 2）获得语义分组——哪些像素属于同一个物体
3. 在每个语义组内做 KNN 聚类，把属于同一个物体的大量点压缩成几个"代理点"（proxy）
4. 这样，整个房间可能只需要几百个 proxy token，而不是几万个点

打个比方：如果传统方法是"把整本书逐字念给语言模型听"，那 Proxy3D 就是"给语言模型一个结构化的摘要"——每段话用一两句话概括，保留关键信息的同时大幅缩短长度。

为了让语言模型能理解这些 proxy 的含义，Proxy3D 还设计了一个巧妙的四阶段渐进训练：
- 第一阶段：先用简化的标识符号（类似棋子）教模型认识"物体是什么"
- 第二阶段：教模型理解这些物体的 3D 坐标
- 第三阶段：教模型理解物体之间的空间关系
- 第四阶段：最后才用真正的 3D 场景 proxy 训练

这就像教小孩学空间概念：先认识物体名称，再学"远/近"、"上/下"，最后才能理解复杂的空间场景描述。

此外，他们还构建了 SpaceSpan 数据集（318K 样本），整合了多种 3D 视觉-语言任务的数据，用统一格式支持多任务训练。

## 摘要翻译

> 空间智能对于视觉-语言模型（VLM）理解 3D 世界至关重要。现有方法大多沿用 VLM 中传统的 2D 流程，使用像素对齐的表示作为视觉模态的输入。然而，基于对应的模型通过隐式 3D 场景理解往往难以实现空间一致性，而基于表示的模型虽然利用了 3D 几何先验，但在视觉序列序列化方面缺乏效率。为解决这些问题，我们提出了 Proxy3D，为视觉模态构建紧凑而全面的 3D 代理表示。给定视频帧作为输入，Proxy3D 使用语义和几何编码器提取场景特征，并执行语义感知聚类以获得 3D 空间中的一组代理。我们还整理了 SpaceSpan 数据集，并应用多阶段训练来逐步将这些 3D 代理表示与 VLM 对齐。大量实验证明，Proxy3D 在 3D 视觉问答、视觉定位和通用空间智能基准测试中实现了具有竞争力或最优的性能，同时使用更短的序列来表示视觉信息。

## 引言翻译

> 空间推理是人类智能的一个基本方面。当我们探索一个新场景时，我们的视觉感知将 2D 视觉输入作为 3D 空间信息接收，并进一步解码为语言模态，使我们能够描述空间关系。最近的视觉-语言模型（VLM）和更通用的多模态大语言模型（MLLM）具有类似的感知能力，并假设性地可以达到人类水平的空间智能。同时，Yang 等人得出结论，当前的 MLLM 在自我图像视角的邻域内形成了一系列局部世界模型，而非来自给定视频的统一全局模型。类似地，Cai 等人的定量评估表明，人类水平的空间智能与 MLLM 中的空间智能之间存在巨大差距。
>
> 我们认为空间信息的表示对于在 3D 世界中进行准确高效的推理至关重要。例如，具有对应目标的 VLM 通过匹配和对齐跨图像帧的特征来隐式实现 3D 空间感知。然而，它们学到的表示存在训练数据使用效率低下和空间不一致的问题，导致缺乏全局场景理解和高计算成本。与之不同，基于表示的方法通过利用从预训练视觉编码器获得的 2D 图像特征来显式建模 3D 场景。最近的研究应用了经典表示来建模 3D 世界，如点云等。另一条研究路线旨在开发 3D 世界的统一表示。然而，根本挑战仍然是：如何为 MLLM 构建一个具有准确空间信息的 token 序列，同时最小化其大小？
>
> 在本工作中，我们发现编码的视觉模态具有稀疏的语义分布，因此可以利用潜在空间聚类来语义压缩 3D 场景。我们的方法 Proxy3D 通过聚类的 3D proxy 特征避免了复杂的基于神经网络的序列化。最后，我们提出了迭代对齐方法，在训练阶段使用 SpaceSpan 数据集来使压缩的 3D 表示与语言模型适配。我们的主要贡献如下：
>
> - 我们整理了一个 318K 的 SpaceSpan 数据集，具有统一的数据格式，整合了异构视觉信息。
> - 我们提出了 Proxy3D，一种聚合紧凑而全面的空间推理表示的方法。
> - 我们引入了多阶段训练流程，通过数据高效的表示对齐逐步提升 MLLM 的 3D 场景理解能力。

## 论文总结

Proxy3D 提出了一种用于视觉-语言模型的高效 3D 表示方法。给定视频帧输入，该方法利用语义编码器和几何预测器提取场景特征，然后通过语义感知聚类将大量像素级特征压缩为少量 3D 代理（proxy），并结合 BFS 排序和 3D 位置编码注入空间先验。配合精心设计的四阶段渐进训练和 318K 规模的 SpaceSpan 数据集，Proxy3D 仅用 700 个 token 就在多个 3D 空间智能基准上达到了与使用 8000 token 的对应式方法相当甚至更优的性能。

## 核心问题

这篇论文要解决的核心问题是：**如何为视觉-语言模型构建既紧凑又全面的 3D 场景表示**。

这个问题重要是因为空间智能是 VLM 理解物理世界的关键能力，而现有方法存在两个根本性缺陷：

1. **对应式方法**（如 3DRS、Video-3D LLM）：通过帧间匹配隐式学习 3D，但存在空间不一致性问题，且训练数据利用效率低，通常需要 8000 个视觉 token
2. **表示式方法**（如 LLaVA-3D、LEO-VL）：利用点云等 3D 几何先验，但序列化效率低——一个场景的点云序列可能很长，而朴素序列化无法有效建模复杂空间关系

核心观察是：编码后的视觉特征具有**稀疏的语义分布**，即大量像素实际上属于少数几个语义类别，因此可以通过聚类大幅压缩。

## 方法详解

### 整体框架

Proxy3D 的流程分为三个主要阶段：特征提取 -> 语义聚类压缩 -> 多阶段对齐训练。

### 1. 特征提取

输入为 N 帧 RGB 图像 {I_i}，分辨率 H x W x 3。

- **2D 视觉编码器**（基于 Qwen2.5-VL）：提取特征图 {F_i}，大小为 H' x W' x C（H' = H/q, W' = W/q，q 为 patch size）
- **几何预测器**（VGGT）：从图像帧预测点图 {P_i}，大小为 H x W x 3，提供 3D 空间坐标
- **2D 分割模型**（SAM 2）：提取像素级分割掩码 {M_i}，大小为 H x W，标识每个像素属于哪个物体

三者对齐后，得到 patch 级别的三元组 {f_j, p_j, m_j}，其中每个元素同时携带语义特征、3D 坐标和语义标签。

### 2. 语义感知聚类（Semantic-aware Clustering）

这是 Proxy3D 最核心的创新。

**语义分组**：首先按分割掩码的语义标签 g 对所有 patch 分组：
G_g = {f_j, p_j | m_j = g}

每个组 G_g 包含属于同一物体（或同一语义类别）的所有特征和坐标。

**组内 KNN 聚类**：对每个语义组用 K-nearest neighbors 聚类，将组内多个 patch 压缩为 K_g 个 proxy：
{C_{g,j}} = KNN(G_g, K_g)

**动态 proxy 分配**：K_g 根据各语义组在整体序列中的比例动态分配（K_g ∝ |G_g| / L），确保每个组至少分配到一定数量的 proxy，避免遗漏小物体。

最终，一个有 L 个 patch 的场景被压缩为 K << L 个 proxy，每个 proxy 是一个包含语义特征 z_{g,j} 和 3D 坐标 c_{g,j} 的元组。

### 3. 序列化与位置编码

**BFS 排序**：对 3D 组中心进行广度优先搜索遍历，从离原点最近的 3D 分段的根节点开始序列化。空间上相近的物体在序列中也是邻居，帮助 LLM 更准确地捕获空间关系。

**3D 空间位置嵌入**：
- 垂直维度用旋转位置编码（RoPE）
- 水平维度（宽度和长度）用可学习的 Fourier 嵌入
- 两者结合为每个 proxy 注入 3D 空间先验

### 4. 物体引用机制（Identifier Embeddings）

定义 100 个标识符和 213 个物体类别。每个物体有两种嵌入：
- **标识符嵌入**（identifier embedding）：将 <OBJXXX> token 映射为视觉嵌入，实现准确的物体引用
- **语义嵌入**（semantic embedding）：用 Stable Diffusion 生成简化的语义符号图像，经视觉编码器编码后作为类别的视觉表示

这两种嵌入通过加法融合注入到 proxy 序列中，以类似"棋子"的简化表示来桥接空间表示与 LLM 的潜在序列。

### 5. 多阶段渐进训练

四阶段训练，从简单到复杂：

| 阶段 | 内容 | 数据量 | 训练时间 |
|------|------|--------|----------|
| Stage 1: 语义与标识符嵌入 | 用简化视觉输入（标识符+语义图像）训练，教模型识别物体 | 30K | ~2h |
| Stage 2: 坐标对齐 | 训练 3D RoPE 嵌入，教模型理解空间坐标 | 30K | ~2h |
| Stage 3: 空间关系 | 用 115K 物体间关系问题训练空间推理 | 115K | ~3h |
| Stage 4: Proxy 表示 | 用完整 3D 场景 proxy 和全部 318K 数据训练 | 318K | ~55h |

训练目标为自回归负对数似然损失：
L(θ) = -Σ log P_θ(t_i | t_{<i}, Z)

其中 Z 是 3D proxy 序列，t_i 是输出 token。

### 6. SpaceSpan 数据集

318K 高质量训练数据，包括：
- 155K 来自常见 3D 数据集（ScanQA, ScanRefer, Scan2Cap, SQA3D, Multi3DRefer）
- 163K 来自 MMScan 和 SR-91K
- 115K 物体-物体关系问题（来自 MMScan）

统一数据格式，整合了 QA、视觉定位、密集描述等多种任务类型。

## 实验与可视化

### 实验设置

- **骨干网络**：Qwen2.5-VL-7B
- **输入**：32 帧均匀采样图像，分辨率 512x512
- **几何预测器**：VGGT
- **分割模型**：SAM 2
- **序列长度**：K = 450-1000（默认 700）
- **GPU**：8x A6000

### 3D 问答（ScanQA / SQA3D）

| 模型 | 视觉 token 数 | ScanQA EM | SQA3D EM |
|------|--------------|-----------|----------|
| 3DRS (correspondence) | 8000 | 30.3 | 60.6 |
| Video-3D LLM | 8000 | 30.1 | 58.6 |
| LLaVA-3D (representation) | 3096 | 27.0 | 55.6 |
| LEO-VL | 750 | 22.6 | 60.8 |
| **Proxy3D** | **700** | 25.2 | 57.5 |

Proxy3D 仅用 700 token 就达到了与 8000 token 的对应式方法相当的性能。

### 3D 视觉定位（ScanRefer / Multi3DRefer）

Proxy3D 在 ScanRefer 和 Multi3DRefer 上取得了 SOTA 或次优结果：
- ScanRefer Uni@0.5: 84.0（最优）
- Multi3DRefer Acc@0.5: 54.1

显著优于 Chat-Scene（82.5/50.2）和 Descrip3D（83.2/51.8）等基于物体提议的方法。归因于语义分组带来的准确物体区分能力。

### 空间推理（VSI-Bench）

| 模型 | Obj.Cnt. | Abs.Dist. | Obj.Size | RoomSize | Rel.Dist. | Rel.Dir. | Avg. |
|------|----------|-----------|----------|----------|-----------|----------|------|
| Human | 94.3 | 47.0 | 60.4 | 45.9 | 94.7 | 95.8 | 79.2 |
| Spatial-MLLM | 65.3 | 34.8 | 63.1 | 45.1 | 41.3 | 46.2 | 48.4 |
| **Proxy3D** | 63.9 | 41.9 | 67.2 | 42.8 | 50.3 | 46.5 | 47.0 |
| Qwen2.5-VL-7B | 40.9 | 14.8 | 43.4 | 10.7 | 38.6 | 38.5 | 33.0 |

Proxy3D 排名第二（仅次于依赖 GRPO 后训练的 Spatial-MLLM），在物体计数和尺寸测量上接近甚至超过人类水平。

### 消融实验关键发现

1. **语义分组**：对视觉定位影响巨大（+20 点 Acc@0.5），对 QA 影响中等。无语义分组的朴素聚类导致物体引用不准确。
2. **坐标对齐**：显著提升房间尺寸估计、路线规划和外观顺序任务。
3. **帧间交叉注意力**：Proxy3D 对此高度鲁棒，即使没有帧间视觉关系也能建立场景理解，这与严重依赖帧间相似性的对应式方法形成鲜明对比。
4. **序列长度**：K=450/700/1000 性能逐步提升，但 K=700 是较好的复杂度-精度平衡点。
5. **动态 proxy 分配**：5 个 proxy/物体效果最佳。过多 proxy 会引入噪声。

## 相关工作

### 对应式方法
- **3DRS**：通过多视图图像对应发展 3D 意识
- **Video-3D LLM**：通过视频帧和 3D 位置编码实现对齐
- **GPT4Scene**：用鸟瞰图辅助 3D 理解
- **Ross3D**：引入显式视图重建注入 3D 意识
- **Proxy3D 的区别**：这些方法需要 8000 token，存在空间不一致性；Proxy3D 通过显式 3D proxy 表示和语义聚类仅需 700 token

### 表示式方法
- **LLaVA-3D**：使用 3D 几何先验，但序列长度 3096
- **LEO-VL**：750 token + SceneDPO 后训练
- **Spatial-MLLM**：3096 token + GRPO 后训练
- **PQ3D / VLM-3R / LEO-VL**：开发统一表示跨越几何先验、实例特征和全局属性
- **Proxy3D 的区别**：专注于通过语义感知聚类学习紧凑 proxy 表示，在保持表示式方法优势的同时最小化序列长度

### 3D 数据集与基准
- ScanQA、SQA3D、ScanRefer、Multi3DRefer、Scan2Cap 用于评估
- VSI-Bench 用于空间推理评估
- MMScan 提供丰富的空间推理数据
- SpaceSpan 整合以上数据，统一格式，新增物体关系数据

## 潜在未来工作

1. **结合强化学习后训练**：消融实验表明 Proxy3D 未使用 GRPO 等后训练方法，而 Spatial-MLLM 通过 GRPO 获得了额外提升。将奖励学习与 Proxy3D 结合可能进一步提升性能。

2. **扩展到更大模型**：当前基于 7B 模型验证，扩展到 72B 或更大模型可能释放更多潜力。

3. **动态自适应 proxy 数量**：当前使用固定或比例分配的 proxy 数量，可以探索基于场景复杂度自适应决定每个物体的 proxy 数量。

4. **流式处理与实时推理**：论文展示了对帧间注意力的鲁棒性，可以进一步探索流式视频输入下的实时 3D 场景理解。

5. **弥合空间推理差距**：VSI-Bench 上与人类水平的差距仍然显著（尤其是外观顺序和路线规划任务），需要新的训练策略或架构设计。

6. **跨场景泛化**：虽然在 ScanNet++ 和 ARKitScenes 上展示了泛化能力，但在更复杂、更大规模的室内外场景中的表现有待验证。

7. **与 3D 生成/规划结合**：紧凑的 proxy 表示可以作为机器人操作和 3D 场景生成的条件输入，将空间理解与下游任务连接。
