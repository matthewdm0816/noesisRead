## 费曼讲解

想象你站在一个大房间里，有人问你"桌上那部手机是什么颜色的？"。你不会随机扫视整个房间——你会先看桌子的方向，然后聚焦到手机附近。KeyVT 做的就是这件事，但它发生在两个层次上。

**第一层（KeyV）：选对"看哪里"。** 现有方法通常用"语义相似度"来挑视角——问题提到"手机"，就挑和"手机"最相关的几张图。但问题是，3D 场景中物体是有空间关系的：手机旁边的杯子、后面的墙壁，可能都是回答问题的关键线索。KeyV 的做法是：先用相机参数算出每张视角之间的"空间距离"（位置 + 朝向），把整个场景切成若干"子场景"（就像把房间分成几个区域），然后根据问题的相关性和子场景的大小，自适应地给每个区域分配不同的视角数量。相关性高的区域多看几张，不相关的少看。

**第二层（KeyT）：选对"看什么细节"。** 选好的视角之间往往有很大的重叠——比如你看了桌子左边、桌子正面、桌子右边，三张图里桌子占了大部分像素。如果 VLM 的输入 token 预算有限，把这些重复信息压缩掉，就能腾出空间放更多有用的细节。KeyT 用最优传输（Optimal Transport, OT）来做这件事：把所有视角的 token 当作一个"源分布"，学习一组数量更少的"虚拟 token"作为"目标分布"，让虚拟 token 尽可能覆盖源分布的所有信息。最终通过 OT 的传输方案（transport plan）选出最有代表性的实际 token。

打个比方：KeyV 像是拍照时选角度——先决定拍哪些方向；KeyT 像是后期裁剪——从每张照片里只保留最有信息量的区域。两层结合，用更少的 token 传递了更多的 3D 信息。

## 摘要翻译

> 近期，利用 2D 视觉-语言模型（VLM）进行零样本 3D 场景理解的研究兴趣日益增长，这得益于其出色的空间推理能力。典型做法是从 3D 点云中采样多个 2D 视角，输入预训练的 VLM 来回答问题。这一范式凸显了输入上下文质量的关键作用，同时也带来了在有限输入预算下尽可能保留任务相关 3D 细节的挑战。我们提出 KeyVT，一种在视角和 token 两个层次上进行输入上下文收集的分层方法。具体而言，我们将像素特征与相机参数相结合，基于语义内容和几何位置评估视角重要性，从而得到空间一致且任务相关的视角。此外，我们通过在最优传输（OT）框架下识别代表性 token 来解决所选视角间 patch 的冗余问题，其中视角 token 和关键 token 被构建为嵌入空间中的两个离散分布。这些关键 token 通过最小化 OT 距离来覆盖所有视角特征。我们在三个广泛使用的基准上评估了该框架，展示了相对于现有无调优方法的显著提升，以及与基于训练的方法相当的性能。

## 引言翻译

> 在处理大规模视觉-文本对之后，视觉-语言模型（VLM，如 GPT-4o 和 Qwen3-VL）在各种多模态任务中展现了出色的能力，包括图像/视频问答、描述和推理。受这些成就以及 Tesla 全自动驾驶系统中纯视觉方案成功的启发，近期研究开始通过从 3D 点云中采样多个视角，将 2D VLM 扩展到 3D 物理世界。一旦获得视角序列，2D VLM 便能自然地对空间环境进行推理，展现出相较于 3D 语言模型的显著潜力。后者通常依赖大量高质量的 3D-文本数据（如点云配详细描述）来对齐几何结构与语言，但此类标注的 3D 数据仍然稀缺，限制了其可扩展性和实际适用性。
>
> 因此，输入上下文的质量在 3D 场景理解中扮演着核心角色。以 3D 问答（3D-QA）为例，理想输入应尽可能多地包含任务相关的 3D 细节，如主要物体及其周围环境。早期研究直接采用均匀采样策略选择输入视角。虽然简单，但这些方法往往引入大量噪声。为提升输入质量，各种关键视角选择方法被提出，包括基于语义相似度的检索、最大化有用信息、时序片段选择、以及训练额外的关键视角选择器。尽管取得了成功，现有方法大多聚焦于查询-视角相关性，可能忽略了查询中未明确提及的关键证据，从而降低下游性能。
>
> 更重要的是，先前工作通常将选定的关键视角以固定数量（如 8 或 16 个视角）送入 VLM，受限于输入预算。直觉上，这些关键视角往往共享语义信息，在区域间存在显著冗余。去除此类冗余通常不会损害预测性能，实际上还能释放输入容量，允许在不增加计算开销的情况下纳入更多关键视角。这一原理与视频理解任务中的 token 压缩技术一致，后者通常过滤时间冗余 token 或聚类特征以提取代表性信息。虽然有效，但这些基于聚类的模型对密集特征敏感，可能无法捕获关键视角间的多样化 token。
>
> 为克服这些限制并在预定义输入预算内收集尽可能多的任务相关信息，我们提出分层框架 KeyVT，首先选择空间一致、任务相关的关键视角，然后以无监督方式压缩这些视角中的代表性 token。超越基于相似度的视角采样方法，我们为每个视角引入几何参数来建模空间关系。给定每个视角的相机参数，我们估计从第一个视角到当前视角的相对距离，这捕获了整个 3D 世界中的空间关系，并能够分割为空间一致的子场景。给定固定数量的关键视角，我们设计自适应策略，根据每个子场景的重要性确定其最优选取视角数量。
>
> 超越视角级选择，我们进一步探索关键视角间的 token 冗余。我们引入"虚拟 token"的概念，其数量少于关键视角 token，但仍能在特征空间中覆盖 3D 细节。与先前基于聚类的模型不同，我们提出在最优传输（OT）理论下学习虚拟 token。OT 提供了在给定代价函数下将质量从一个分布传输到另一个分布所需代价的原则性度量。具体而言，所有关键视角 token 被表示为编码任务相关证据的冗余集合，虚拟 token 集合旨在通过从冗余集合传输特征来总结关键信息。利用 OT 在概率分布空间中建模几何结构的能力，我们的模型能够发现多样且具代表性的虚拟 token。重要的是，OT 学到的传输方案度量了从虚拟 token 到视角 token 的概率值，这提供了一种简单有效的方式，通过排序传输方案来识别关键 token。

## 论文总结

本文提出 KeyVT，一种分层的零样本 3D 问答框架，通过几何感知的关键视角选择（KeyV）和最优传输引导的关键 token 压缩（KeyT），在有限的 VLM 输入预算下最大化保留任务相关的 3D 信息，在 ScanQA、SQA3D 和 VSI-Bench 三个基准上显著超越现有无调优方法，性能接近甚至超过需要训练的方法。

## 核心问题

**解决什么问题：** 如何在有限的 VLM 输入 token 预算下，为 3D 场景问答任务选择最优的输入上下文（视角 + token）。

**为什么重要：** 3D 场景通过采样多个 2D 视角输入 VLM，但 token 预算远不足以覆盖所有视角的所有 token。输入上下文的质量直接决定下游问答性能。

**现有方法不足：**
1. **视角选择**：现有方法主要基于查询-视角语义相似度，忽略了空间几何信息和周围环境上下文
2. **token 压缩**：现有聚类方法（如 FLoC、DivPrune）对密集特征敏感，可能丢失多样性 token
3. **视角级冗余**：选定的关键视角之间存在大量重叠区域，直接送入 VLM 浪费 token 预算

## 方法详解

### 整体框架

KeyVT 是一个两阶段分层框架：
1. **KeyV**（Key View）：从全部 2D 视角中选择空间一致、任务相关的关键视角
2. **KeyT**（Key Token）：从关键视角的 token 中去除冗余，选出代表性 token

### KeyV: 几何感知的关键视角选择

**步骤 1 — 空间距离计算：**
定义视角间距离，综合考虑位置和朝向：

$$D(V_0, V_i) = \|C_0 - C_i\| + \theta(R_0, R_i)$$

其中 $C = -R^\top t$ 是世界坐标系下的相机中心，$\theta(R_0, R_i) = \arccos\left(\frac{\text{tr}(R_0^\top R_i) - 1}{2}\right)$ 是朝向角距离。

**步骤 2 — 子场景分割：**
用窗口大小 $\delta$ 对距离序列 $D$ 进行窗口分割，得到子场景序列 $\{L_1, L_2, \ldots, L_{|L|}\}$。每个子场景内的视角空间上相近，形成空间一致的区域。

**步骤 3 — 自适应视角分配：**
对每个子场景计算语义相关性分数：

$$r_l = \max(O_l) + \text{mean}(O_l), \quad O_{l,i \in L_l} = \text{BLIP2}(V_i, Q)$$

结合最大值（捕获最相关视角）和均值（保证整体覆盖）。然后按重要性分配关键视角数量：

$$N_l = \left\lfloor K \cdot \frac{W_l}{\sum_{l'} W_{l'}} \right\rfloor, \quad W_l = r_l \cdot \sqrt{|L_l|}$$

平方根缩放防止过大的子场景主导分配。最后在每个子场景内按相关性排序选 Top-$N_l$ 个视角。

### KeyT: OT 引导的关键 Token 选择

**步骤 1 — 构建分布：**
将关键视角的所有 token 嵌入集合 $P = \{e_1, \ldots, e_N\}$ 作为源分布，虚拟 token 集合 $Q = \{c_1, \ldots, c_M\}$（$M < N$）作为目标分布，均表示为嵌入空间上的离散分布。

**步骤 2 — OT 优化：**
定义代价矩阵为余弦距离 $C_{n,m} = 1 - \cos(e_n, c_m)$，使用 Sinkhorn 距离（熵正则化 OT）：

$$d_{C,\gamma}(P, Q) = \min_{T \in U_\gamma(\alpha, \beta)} \langle T, C \rangle$$

通过 Adam 优化器对虚拟 token $Q$ 进行 10-15 次迭代更新（学习率 $10^{-3}$），最小化 OT 距离。

**步骤 3 — Token 映射：**
虚拟 token 不直接送入 VLM（它们不是真实 patch token），而是通过传输方案 $T$ 找到每个虚拟 token 对应的最近邻真实 token：

$$\text{Nei}(c_m, T) = \{e_i \mid i \in \text{Top-K}(S/M, T_{:,m})\}$$

最终将选出的真实 token 拼接作为 VLM 输入。

### 关键技术创新点

1. **几何感知视角选择**：首次将相机参数（位置 + 朝向）引入 3D QA 的视角选择，超越纯语义相似度方法
2. **自适应子场景分配**：根据子场景重要性和大小自适应分配视角数量
3. **OT 引导的 token 压缩**：用最优传输理论（而非聚类）选择代表性 token，更好地保持多样性
4. **虚拟 token 到真实 token 的映射**：通过 transport plan 将 OT 优化结果落地为可用的 VLM 输入

### 训练/推理流程

- 完全无训练（tuning-free），仅需 BLIP2 编码器做语义相似度计算
- OT 优化仅需 10-15 次迭代，非常轻量
- 在单张 48G RTX 6000ada GPU 上运行
- 可作为即插即用模块适配不同 VLM 后端

## 实验与可视化

### 实验设置

**数据集：**
- ScanQA：基于 ScanNet 的室内 3D QA，约 41k QA 对
- SQA3D：具身场景理解，33.4k 问题，含 agent 位置/朝向上下文
- VSI-Bench：5000+ QA 对，来自自中心视频，涵盖 8 个空间推理子任务

**基线：**
- 3D-LLM 类：ScanQA, SQA3D, 3D-Vista, 3D-LLM, LL3DA, Chat-Scene, 3D-LLaVA
- 训练方法：cdViews, Video-3D LLM
- 关键帧选择：AKS
- Token 压缩：FLoC, DivPrune

**指标：** CIDEr, BLEU-1/4, METEOR, ROUGE-L, EM@1（ScanQA/SQA3D）; MCA, NA（VSI-Bench）

**VLM 后端：** LLaVA-OneVision-7B, LLaVA-Video-7B, Qwen2.5-VL-7B

### 主要结果

**ScanQA/SQA3D（Table 1）：**
- KeyVT 在所有 tuning-free 方法中在 CIDEr 和 EM@1 上表现最佳
- LLaVA-Video-7B + KeyVT 达到 CIDEr 100.7 / EM@1 57.9，与训练方法 Video-3D（102.1 / 58.6）相当
- 相比 AKS（CIDEr 99.0）和 FLoC（99.4），KeyVT（100.7）有明显提升

**VSI-Bench（Table 2）：**
- KeyVT 在三个 VLM 后端上均取得最佳平均性能（33.9, 34.6, 37.0）
- 在多个子任务上表现均衡，显示方法的鲁棒性

### 消融实验

**KeyV 组件消融（Table 5）：**
- 去掉几何感知设计 → CIDEr 从 100.7 降到 93.8（下降最多）
- 去掉子场景分割 → 降到 99.8
- 去掉相关性评分 → 降到 99.4

**KeyT 有效性（Table 6）：**
- 任何视角选择方法 + KeyT 都比不加 KeyT 好
- KeyV + KeyT 组合达到最优

**Token 压缩率（Table 7）：**
- 在 25%/50%/75% 保留率下，KeyT 始终优于 DivPrune 和 FLoC
- 50% 保留率下 KeyT（CIDEr 69.6）接近 100% 基线（70.2）

**相机参数鲁棒性（Table 4）：**
- 5% 噪声下 KeyVT 仍然优于所有基线
- 使用 VGGT 估计的相机参数（100.0）接近真实参数（100.7）

**超参数敏感性：**
- 窗口大小 $\delta$：1.0 最优，0.5-1.5 范围内鲁棒
- OT 迭代次数：10 次最佳，5-15 次均可接受
- 学习率：$10^{-3}$ 最优

### 可视化

- 将选中的关键视角和关键 token 投影回 3D 点云，可观察到：
  - 关键视角形成空间一致的子场景（如围绕"水槽"或"架子"的区域）
  - 关键 token 保留了主要几何和语义结构，同时去除冗余
- t-SNE 可视化显示选出的关键 token 广泛覆盖了原始 token 的分布

## 相关工作

**3D 大语言模型（3D-LLM）：**
- 如 PointLLM, LL3DA, 3D-LLM, Chat-Scene, 3D-LLaVA 等，需要大量 3D-文本配对数据进行训练
- KeyVT 是 tuning-free 的，不需要 3D 训练数据，直接利用预训练 2D VLM

**关键帧选择：**
- AKS：自适应关键帧采样，基于时序连续性
- BOLT：逆变换采样提高帧多样性
- Q-Frame：动态分辨率的查询感知选择
- KeyVT 区别：显式利用相机参数做几何感知选择，而非仅靠时序或语义相似度

**Token 压缩：**
- DivPrune：基于多样性的 token 剪枝
- FLoC：基于子模设施位置函数的 token 压缩
- LLaVA-Prumerge：自适应 token 合并
- KeyVT 区别：用 OT 理论而非聚类，更好地保持 token 多样性

## 潜在未来工作

1. **端到端联合优化**：当前 KeyV 和 KeyT 是顺序执行的，可以探索联合优化视角选择和 token 压缩
2. **查询感知的 token 压缩**：当前 KeyT 不依赖查询信息，引入查询感知可能进一步提升性能
3. **扩展到更多 3D 任务**：如 3D grounding、3D 导航、3D 描述生成等
4. **动态预算分配**：根据问题难度自适应调整输入 token 预算
5. **与 3D-LLM 结合**：将 KeyVT 的视角/token 选择能力与 3D-LLM 的几何理解能力结合
6. **处理密集杂乱场景**：论文承认在密集杂乱或模糊空间关系场景下仍有失败案例，可探索更好的处理策略
