## 费曼讲解

想象你拿着手机绕着房间走一圈拍了个视频，然后问 AI："房间里有几张椅子？沙发在哪里？"——这就是我们想让 3D 多模态大语言模型做到的事情。

现有的方法要么需要你先用专业设备扫描出点云，要么需要深度相机拍出的 RGBD 数据，再把这些 3D 数据喂给模型。这就好比你必须先建好一栋房子的 BIM 模型，才能让设计师做室内设计——门槛太高了。

我们的核心想法其实很简单：**视频本身就包含了丰富的 3D 信息**。当你拿着手机走一圈，不同角度的画面已经隐含了深度、结构和空间关系。问题是，怎么把这些隐含的几何信息和语言理解能力有效地结合起来？

我们设计了一个叫 **Cross-Task Adapter (CTA)** 的模块，它就像一个"翻译官"。想象有两个人——一个擅长看空间结构（几何专家），一个擅长理解语言（语义专家）。CTA 里有一些"桥接 token"，它们分别去"听"几何专家和语义专家说了什么，然后把两边的信息融合起来。这样，几何专家会告诉语义专家"这个物体距离你 2 米"，语义专家也会反过来帮助几何专家"那是一个沙发，不是一堵墙"。

还有一个关键问题：从视频重建出来的深度是"相对深度"——你只知道 A 比 B 远，但不知道具体多远。这就像你用眼睛看东西知道远近，但说不出精确距离。我们设计了一个 **Metric Depth Model**（度量深度模型），它能从相对深度中恢复出真实的物理尺度。有了真实尺度，模型才能告诉你"那个桌子长 1.2 米"而不是"那个桌子大概比椅子远一点"。

最后，训练策略也很关键。我们采用两阶段蒸馏：第一阶段用两个"老师"（一个几何老师 VGGT，一个语义老师 LLaVA-3D）教共享编码器同时学几何和语义；第二阶段把所有模块联合微调。这就像先分别跟数学老师和语文老师打好基础，再做综合练习。

## 摘要翻译

> 近期多模态大语言模型（MLLMs）的发展显著提升了二维领域的视觉-语言（VL）推理能力。然而，将这些能力扩展到三维场景理解仍然是一个重大挑战。现有的三维多模态大语言模型（3D-MLLMs）通常依赖三维数据输入，这限制了其可扩展性和泛化能力。为了解决这一局限，我们提出了 Vid-LLM，一种基于视频的 3D-MLLM，能够直接处理视频输入而无需外部三维数据，使其在实际部署中更加实用。在我们的方法中，几何先验被直接用于提升场景感知性能。为了将几何线索紧凑地整合到 MLLM 中，我们设计了一个跨任务适配器（CTA）模块，用于对齐三维几何先验与视觉-语言表示。为确保几何一致性和完整性，我们引入了一个度量深度模型，从重建输出中恢复真实尺度的几何信息。最后，模型通过两阶段蒸馏优化策略进行微调，实现快速收敛并稳定训练。在多个基准上的大量实验验证了我们方法在三维问答、三维密集描述和三维视觉定位任务上的有效性，展示了出色的多任务能力。

## 引言翻译

> 大语言模型（LLMs）和多模态大语言模型（MLLMs）的近期进展强化了语言作为通用接口的范式，显著改善了跨模态感知与推理。将这一进展扩展到三维领域，近期研究聚焦于三维感知多模态大语言模型（3D-MLLMs），它们在语言接口下统一了三维场景理解与视觉-语言推理。这一研究方向凸显了将语言锚定在持久化三维空间表示中的重要性，为系统化的场景级推理提供了统一路径。
>
> 近期研究在三维视觉-语言（3D VL）推理方面取得了重大进展，但大多数方法依赖复杂的三维输入，在数据收集、预处理和计算方面产生高昂成本。部分模型依赖点云或带有渲染视图或语义-几何特征的重建场景，而其他模型虽然采用更简单的输入，但仍依赖显式的三维场景表示，如与语义表示对齐的重建物体。尽管这些方法有效，但它们依赖深度、姿态或外部模块，导致大量的数据和工程开销以及较高的内存和延迟成本。这种刚性的输入要求和系统复杂性从根本上限制了当前 3D-MLLMs 的可扩展性和可迁移性。
>
> 为了克服这些局限，一个更通用的解决方案是使模型能够直接从视频重建场景几何，从而消除对外部深度、姿态或配准模块的依赖。更重要的是，重建和推理本质上是相互依赖的：几何结构支撑语义理解，而语义推理反过来提供上下文先验来指导和优化几何建模。
>
> 在本工作中，我们介绍了 Vid-LLM，一个紧凑的模型，能够从单目视频输入中联合执行重建和三维视觉-语言推理。Vid-LLM 的核心组件是一个跨任务适配器（CTA），它紧密耦合重建与推理，实现内在的几何-语义交互及相互增强与约束。CTA 解耦了几何感知和语言感知特征；几何流随后由全局-帧注意力骨干和专用头部处理，用于估计相机姿态和相对深度，再通过度量深度模型进行真实尺度校准。恢复的三维信息随后与语义特征融合构建三维 patch，输入 LLM 进行空间推理。最后，两阶段训练策略确保收敛并提升整体性能。在多个三维视觉-语言基准上的大量实验展示了 Vid-LLM 的性能，并证实了其作为基于视频的三维多模态推理的实用且可扩展框架的有效性。
>
> 我们的主要贡献总结如下：
> - 我们提出了 Vid-LLM 用于多功能三维场景理解。该框架不依赖密集三维输入或先验姿态，使其在实际部署中更加实用。
> - 我们设计了跨任务适配器来对齐三维几何先验与视觉-语言表示，增强了三维视觉几何先验向 MLLM 的整合。进一步采用两阶段训练策略以提高稳定性和性能。
> - 在真实数据集上进行了广泛的实验评估。实验结果表明，我们的方法在问答、密集描述和视觉定位方面均取得了优越性能。我们将发布代码以促进交流。

## 论文总结

本文提出了 Vid-LLM，一种基于视频输入的紧凑型三维多模态大语言模型，通过跨任务适配器（CTA）将三维重建分支与视觉-语言推理分支紧密耦合，实现了仅从单目视频即可完成三维问答、三维密集描述和三维视觉定位等任务，并在多个基准上达到或接近使用显式三维数据输入的方法的性能。

## 核心问题

**问题**：现有的 3D-MLLMs 严重依赖点云、重建场景、深度图等显式三维数据输入，这导致数据采集成本高、预处理复杂、计算开销大，从根本上限制了模型的可扩展性和可迁移性。

**为什么重要**：在真实世界部署中，我们最容易获取的输入形式是视频（如手机拍摄），而非专业设备采集的三维数据。如果模型能直接从视频理解和推理三维场景，将极大地降低使用门槛。

**现有方法的不足**：
1. 基于点云/重建场景的方法（如 3D-LLM、LEO、ChatScene）需要完整的三维重建流水线作为前置
2. 部分方法虽然使用视频输入但仍依赖外部 3D 模块生成显式几何表示（如 3DRS†）
3. 简单地将重建几何拼接到 LLM 输入（concatenation pipeline）无法有效利用几何信息进行推理——实验表明 VGGT+LLaVA-3D 的 3D VL 性能远低于端到端方法
4. 相对尺度几何无法提供可靠的推理线索，需要度量尺度对齐

## 方法详解

### 整体框架

Vid-LLM 的架构由以下核心组件构成：
1. **共享视觉编码器**（DINOv2）：从输入视频帧提取基础视觉 token
2. **跨任务适配器（CTA）**：将共享特征分解为几何流和语义流，并通过桥接 token 实现双向信息交换
3. **3D 重建分支**：预测相机姿态、相对深度，通过度量深度模型恢复真实尺度
4. **3D 视觉-语言推理分支**：将重建几何与语义特征融合构建 3D patch token，输入 LLM 进行推理
5. **两阶段训练策略**：双教师蒸馏 + 联合优化

### 跨任务适配器（CTA）

这是本文最核心的创新。给定 DINOv2 提取的基础 token $T_{\text{base}} \in \mathbb{R}^{N \times C}$：

**特征投影**：通过两个轻量 MLP 投影头将共享视觉 token 映射到不同的任务空间：
$$T_{\text{geom}} = \phi_{\text{geom}}(T_{\text{base}}), \quad T_{\text{lang}} = \phi_{\text{lang}}(T_{\text{base}})$$

**桥接机制**：引入可学习的桥接 token $T_{\text{bridge}} \in \mathbb{R}^{K \times C}$（默认 $K=16$），它们作为共享记忆单元分别 attend 到几何和语义特征：
$$T'_{\text{bridge}} = \text{Attn}(T_{\text{bridge}}, T_{\text{geom}}^{\text{fused}}, T_{\text{geom}}^{\text{fused}}) + \text{Attn}(T_{\text{bridge}}, T_{\text{lang}}^{\text{fused}}, T_{\text{lang}}^{\text{fused}})$$

更新后的桥接 token 再注入回两个特征流，实现几何-语义的双向交互。

**设计优势**：相比直接拼接或交叉注意力，桥接 token 提供了一个稳定的共享隐空间，使几何线索能一致地传递给语义特征，反之亦然。

### 3D 重建模型

**几何编码与预测头**：基于 CTA 增强的几何特征 $T'_{\text{geom}}$，加上相机 token 和注册 token，通过 Global-Frame Attention 骨干（基于 VGGT 架构）产生整合的几何表示，输入两个预测头：
- **Camera Head**：估计相机内外参
- **DPT Head**：预测相对深度图 $\hat{D}_{\text{rel}} \in \mathbb{R}^{H \times W}$

**度量深度模型**：DINOv2 特征通过 DPT 风格解码器产生多尺度深度表示。每个像素的深度通过 bin-based 公式建模：
$$\hat{d}(i) = \sum_{k=1}^{N} p_i(k) \cdot c_i(k)$$

其中 $p_i(k)$ 是第 $k$ 个 bin 的概率，$c_i(k)$ 是自适应精炼的 bin 中心 $c_i(k) = c_k + \Delta c_i(k)$，$\Delta c_i(k) = r_k(F_i)$ 从解码器特征预测。

**真实尺度对齐**：通过加权最小二乘法估计相对深度 $\hat{D}_{\text{rel}}$ 和度量深度 $\hat{D}_{\text{metric}}$ 之间的缩放因子。随机采样 16 张图像计算逐图缩放因子，取中位数作为场景级因子。选择这种对齐策略而非直接使用度量深度作为输出，是因为 DPT 头在纹理细节上比度量深度模型更准确。

### 3D 视觉-语言模型

**3D Patch 构建**：每个 2D 语义特征 $T'_{\text{lang}}(i,j)$ 利用估计的深度 $\hat{D}$、相机姿态 $(\hat{R}, \hat{t})$ 和内参 $K$ 反投影到 3D 空间：
$$P_v(i,j) = \hat{R}^{-1} K^{-1} [i, j, 1]^\top \hat{D}(i,j) - \hat{R}^{-1} \hat{t}$$

3D 坐标通过 MLP 编码为位置嵌入 $P'_v(i,j)$，与语义特征融合：
$$T_{3D}(i,j) = T'_{\text{lang}}(i,j) + P'_v(i,j)$$

这些 3D patch token 送入 LLM（基于 LLaVA 架构）进行空间推理。

### 训练策略

**Stage 1 - 双教师蒸馏**：
- 几何教师：预训练的 VGGT 的 DINO 编码器
- 语义教师：LLaVA-3D 的 CLIP 编码器
- 蒸馏损失：$L_{\text{distill}} = L_{\text{geo}}^{\text{feat}} + L_{\text{lang}}^{\text{feat}} + \lambda L_{\text{sc}}$
  - 几何损失 $L_{\text{geo}}^{\text{feat}}$：patch 级 L2 损失
  - 语义损失 $L_{\text{lang}}^{\text{feat}}$：余弦相似度损失
  - 结构一致性损失 $L_{\text{sc}}$：基于拼接特征的 Gram 矩阵差异（Frobenius 范数），确保几何和语义特征之间的结构关系与教师一致

**Stage 2 - 联合优化**：
$$L_{\text{joint}} = L_{\text{recon-task}} + L_{\text{VL-task}} + L_{\text{MD}}$$

- $L_{\text{recon-task}}$：多任务重建损失（相机 + 深度 + 点云）
- $L_{\text{VL-task}}$：3D VL 推理损失（交叉熵 + 边界框回归 + 匹配损失）
- $L_{\text{MD}}$：度量深度损失，结合全局尺度惩罚和鲁棒局部精炼项：
$$L_{\text{MD}} = b^2 + \frac{1}{K} \sum_{i=1}^{K} \frac{(e_i - b)^2}{1 + \alpha |e_i - b|}$$
其中 $e_i = \log(d_i^{\text{pred}} + \epsilon) - \log(d_i^{\text{gt}} + \epsilon)$ 是对数深度误差。

**关键设计**：3D-VL 推理分支的损失不用于优化 3D 重建模型（单向梯度流），确保 CTA 作为稳定的桥梁。

## 实验与可视化

### 实验设置

- **视觉骨干**：DINOv2-L（24 层 Transformer，隐藏维度 1024）
- **数据处理**：每场景均匀采样 32 帧，短边 resize 到 518，裁剪使长宽为 14 的倍数
- **优化器**：AdamW（$\beta_1=0.9, \beta_2=0.999$，权重衰减 0.05，梯度裁剪 1.0）
- **评估数据集**：
  - 3D QA：ScanQA、SQA3D
  - 3D Dense Captioning：Scan2Cap
  - 3D Visual Grounding：ScanRefer、Multi3DRefer、Nr3D、Sr3D
  - 重建：ScanNet、NYU Depth v2、Co3Dv2、RealEstate10K

### 主要结果

**3D 问答**（ScanQA）：Vid-LLM CIDEr 101.9，在 video-based 模型中最佳，超过 3DRS†（94.7）11%，接近最好的 3D-based 模型 LLaVA-3D（103.1）。

**3D 密集描述**（Scan2Cap@0.5）：CIDEr 81.5，在 video-based 方法中最佳，接近 3D-based 最佳的 3DRS（86.1）。

**3D 视觉定位**：
- ScanRefer Acc@0.5：63.2（超越所有 3D-based 和 video-based 方法）
- Multi3DRefer Acc@0.5：56.1（同上）
- Nr3D Overall：65.4（超越 SceneVerse 64.9）
- Sr3D Overall：77.8（超越 SceneVerse 77.5）

**与联合重建-推理模型对比**：
- 相比 concatenation pipeline（VGGT+LLaVA-3D†），Vid-LLM 在所有 3D VL 任务上全面领先，尽管后者重建分数更高（0.591 vs 0.582），说明简单拼接几何信息不足以支撑推理
- 相比 Uni3DR²（需要 GT 相机位姿），Vid-LLM 重建质量略优（0.582 vs 0.580），3D VL 平均提升 47.6%
- 推理速度最快：1.6 s/scene（vs concatenation 的 2.7 s/scene）

**重建性能**：
- 相机姿态估计接近 VGGT 教师模型
- NYU 深度估计 $\delta_1=0.987$，log10=0.010（与 VGGT† 相当）
- ScanNet F-score 0.582（vs VGGT† 0.580）

### 消融实验

**CTA 模块**：
- 无 CTA：ScanQA 33.7、Scan2Cap 36.6、ScanRefer 34.1——共享表示无法同时支撑两个任务
- Concat-SA（拼接后自注意力）：38.2、41.9、39.6——改善有限
- 无 Bridge 的交叉注意力：40.8、45.6、42.2——更好但无稳定共享空间
- 完整 CTA（16 tokens）：45.7、53.2、48.4——最优
- 4/8/16/32 bridge tokens 对比：16 个达到最佳平衡

**度量深度模块**：
- 无度量深度：ScanQA 仅 29.6（几乎失败），尺度歧义破坏了几何一致性
- 有度量深度无对齐：42.1（改善但仍受限）
- 完整方案：45.7

**训练策略**：
- 单阶段训练收敛慢且损失高
- 单教师蒸馏效果有限
- 去掉关系损失收敛更慢
- 完整两阶段策略效果最佳

### 可视化分析

论文展示了 ScanRefer 上的 3D 视觉定位可视化（在重建点云上绘制预测 3D 边界框）以及三种任务的定性结果。挑战案例分析表明：当视频视角有限时（部分物体未被观测），重建不完整导致计数或定位错误；反光表面导致深度估计不稳定；但当 2D 语义线索充足时，CTA 和桥接机制仍能产生准确描述。

## 相关工作

**3D-MLLMs**：现有方法（3D-LLM、LEO、ChatScene、LLaVA-3D、3DRS 等）大多依赖点云或重建场景等显式三维输入。VGLLM 也采用视频输入但使用 VGGT 几何编码器提取 3D 特征。Vid-LLM 的区别在于通过 CTA 实现重建-推理的紧密耦合，而非简单的两阶段流水线。

**3D 重建**：从经典多视图几何（COLMAP）到神经隐式表示（NeRF）再到前馈 Transformer 架构（DUSt3R、MASt3R、VGGT）。Vid-LLM 基于 VGGT 架构构建重建分支，并额外引入度量深度模型恢复真实尺度。

**几何先验在 VLM 中的使用**：现有方法分三类——显式输入注入（深度/点云作为额外模态）、数据和训练层面的内化（空间标注语料/几何蒸馏）、模块化或 prompt-based 集成。Vid-LLM 属于通过视频驱动的重建分支生成几何并实现与语义分支对齐的方式。

## 潜在未来工作

1. **改进重建分支的几何保真度**：论文挑战案例分析表明，重建不完整是主要失败模式。更强的重建能力（更好的遮挡处理、反光表面处理）可以直接提升 3D VL 性能。
2. **扩展到更多 3D VL 任务**：当前覆盖 QA、Captioning、Grounding 三类任务，可扩展到 3D 规划、具身交互等。
3. **更高效的视频编码**：当前均匀采样 32 帧，对于长视频或实时场景可能需要更高效的帧选择或流式处理策略。
4. **泛化到户外场景**：当前主要在室内 ScanNet 场景评估，户外场景的深度范围更大、纹理更丰富，可能需要不同的策略。
5. **与 LLM 的更深整合**：当前将 3D patch token 作为 LLM 输入，可以探索更紧密的整合方式，如在 LLM 中间层注入几何信息。
