## 费曼讲解

想象你正在训练一个机器人助手理解 3D 室内场景。当你说"找到桌子左边的红色椅子"时，有两类方法可以做这件事：

**第一类：符号方法（"程序员"风格）。** 就像写程序一样，把指令拆成一步步操作：先扫描场景里所有物体 -> 过滤出所有"椅子" -> 过滤出所有"红色"的 -> 检查哪些红色椅子在桌子左边。这种方法非常清晰，每一步都能看到中间结果（可解释），但问题是它只能识别预定义的概念（比如只能识别"椅子"，认不出"舒适的家具"这类开放性描述）。

**第二类：多模态大语言模型（"直觉"风格）。** 把 3D 场景和指令一起扔给一个大模型，让它直接给出答案。这种方法能理解任意自然语言，但推理过程是个黑盒——如果答错了，你完全不知道是认错了物体、理解错了空间关系、还是逻辑组合出了问题。

APEIRIA 的核心想法是：**为什么不把两种方法的优势结合起来？** 具体来说，符号程序里蕴含的"推理模式"（如何分解问题、如何逐步验证）是可以从程序中"蒸馏"出来的，然后教给大语言模型，让它用自然语言的 chain-of-thought 来模仿这种逐步推理。

这个蒸馏过程通过一个精心设计的三阶段课程来实现：

1. **先学会看**（感知对齐）：教模型认识 3D 物体的类别、属性和位置，就像教小孩先认识周围的东西。
2. **再学会想**（符号推理注入）：用合成数据集（Sr3D 等）生成带完整中间验证的推理链，教模型像符号程序一样逐步分解和验证。这一步的巧妙之处在于，合成数据天然就有完美的中间监督——每一步的输入输出都是已知的。
3. **最后学会泛化**（开放集泛化）：用强化学习把推理能力扩展到真实世界的开放词汇指令上。由于真实数据没有中间步骤的标注，所以用结果奖励（预测框是否匹配）来引导探索。

打个比方：Stage 2 就像用有标准答案的练习题教会学生解题的"语法"（步骤和方法），Stage 3 则是让学生去做没有标准答案的开放性题目，只告诉他们最终结果对不对，但因为已经学会了"语法"，他们能正确推理。

## 摘要翻译

> 当前的 3D 空间推理方法面临一个根本性权衡：神经符号 3D（NS3D）概念学习器通过组合式程序实现了可解释的推理，但被限制在封闭集概念词汇和简单程序中；端到端 3D 多模态大语言模型（3D MLLMs）能够处理复杂的自然语言和开放词汇概念，但缺乏显式空间验证的黑盒推理。我们提出了 APEIRIA，一个神经符号 3D MLLM，通过将符号推理模式蒸馏到带有自然语言思维链的 MLLM 中来桥接这两种范式。我们的三阶段课程逐步构建推理能力：a）3D 感知对齐将物体视觉-几何特征与 LLM 对齐，b）CoT-SFT 从符号程序轨迹中教会查询分解和逐步验证，c）CoT-RL 将推理模式扩展到开放集概念和深层嵌套指令。通过转移推理模式而非概念特定知识，APEIRIA 保留了 NS3D 的关键优势：透明推理以及规划和感知组件的模块化可互换性。在定位、问答和描述任务上的评估表明，APEIRIA 超越了先前的 NS3D 方法，并在 3D 空间推理数据集上匹配了最先进的 3D MLLMs，统一了符号方法的系统性推理与 MLLMs 的灵活性。

## 引言翻译

> 3D 空间推理是具身 AI 和 3D 场景理解的基础，两种范式已被提出来应对这一挑战（见图 1）。主流范式 3D 多模态大语言模型（3D MLLMs）利用 LLM 的语义能力处理自由形式的指代表达。然而，由于缺乏显式推理结构，这些黑盒模型直接将指令映射为答案，没有可解释的验证步骤。当它们失败时，没有轨迹可以诊断错误发生在物体识别、关系理解还是逻辑组合哪个环节。
>
> 相比之下，另一条较小的研究方向——神经符号 3D（NS3D）学习器——通过将查询分解为模块化程序来追求可解释的空间推理，这些程序由训练好的概念特定网络迭代执行。虽然在封闭集词汇内的系统性泛化方面表现出色，但它们面临两个根本性障碍：（i）刚性的概念特定网络无法处理开放词汇的空间查询（如"舒适的椅子"、"凌乱的桌子"）；（ii）训练模块化组件需要在简单合成数据集之外不可用的密集逐步监督。
>
> 该领域似乎陷入了一种权衡：可解释但有限的 NS3D 学习器对阵强大但不透明的 MLLMs。然而，我们发现了一个机会，可以将概念学习与推理学习解耦。神经符号程序编码了精确、系统性的推理模式（思维的"语法"），这些可以被蒸馏到 MLLM 中，而 MLLM 本身提供了理解 3D 语义的开放世界知识。在像 Sr3D 这样的合成数据集上，指令从启发式规则模板生成，符号程序保证有完整的中间监督：每个原语的输入和输出都可以根据真实标注构建和验证。这为将推理能力转移到 3D MLLMs 提供了理想的跳板：我们可以先将这些经过验证的执行轨迹蒸馏到 MLLM 中，教会模型系统性的查询分解和逐步空间验证技能。然后，在真实世界数据集上，由于此类完整轨迹不可用，我们采用基于结果监督的强化学习来引导和扩展这些已学习的推理模式到开放词汇概念和复杂嵌套指令上。
>
> 基于这一洞察，我们提出 APEIRIA，一个用于 3D 空间推理的神经符号 MLLM，它统一了符号方法的系统严谨性与 LLM 的语义灵活性。APEIRIA 建立在高效的物体中心表示之上，通过基于课程的推理蒸馏框架进行训练，逐步构建空间推理能力。在阶段 1（感知对齐）中，我们教模型"看"；在阶段 2（符号推理注入）中，我们教模型"逐步思考"；在阶段 3（开放集和复杂推理泛化）中，我们教模型适应真实世界指令。这产生了一个端到端模型，能够解决复杂的 3D 空间推理任务，同时保留神经符号方法的关键优势：通过显式推理轨迹实现推理透明性，以及随着更强 LLM 或 3D 感知模型的出现而可互换的规划和感知组件的模块性。
>
> 具体而言，我们的关键贡献是：1）提出了 APEIRIA，一个桥接符号推理和现代 MLLMs 的神经符号 3D MLLM。2）提出了基于课程的推理蒸馏框架。3）证明了 APEIRIA 在 3D 空间推理基准上超越了先前的神经符号方法和最先进的 3D MLLMs。

## 论文总结

APEIRIA 提出了一个神经符号 3D 多模态大语言模型，通过三阶段课程（感知对齐、CoT-SFT 从符号程序轨迹蒸馏推理模式、CoT-RL 扩展到开放集）将符号推理的可解释性和系统性与大语言模型的语义灵活性统一起来，在 ScanRefer、Multi3DRefer 等基准上超越了传统神经符号方法并匹配或超越了最先进的 3D MLLMs。

## 核心问题

**问题**：3D 空间推理领域存在一个根本性权衡——神经符号方法（NS3D）推理可解释但只能处理封闭集词汇和简单程序；端到端 3D MLLMs 能理解开放词汇的复杂语言但推理过程是黑盒，无法诊断错误来源。

**为什么重要**：对于具身 AI（机器人导航、室内助手等），可解释的空间推理至关重要——不仅需要正确回答，还需要知道为什么这样回答，以便调试和信任校准。而现实世界的语言描述充满了开放性词汇（"舒适的"、"凌乱的"），这恰好是符号方法无法处理的。

**现有方法的不足**：
- NS3D 方法（如 NS3D、LARC）依赖概念特定网络，无法处理开放词汇查询；训练需要密集的逐步监督，仅在简单合成数据集上可用。
- 3D MLLMs（如 Chat-Scene、LLaVA-3D）将推理视为黑盒映射，失败时无法定位错误来源（物体识别？关系理解？逻辑组合？）。
- 并发的 RL 方法（3D-R1、Scene-R1）缺乏结构化推理轨迹初始化，RL 探索不稳定且低效。

## 方法详解

### 整体框架

APEIRIA 的核心思想是：**推理模式可以与概念知识解耦**。符号程序编码了推理的"语法"（如何分解问题、如何逐步验证），而 LLM 提供了理解开放世界概念的语义能力。通过将前者蒸馏到后者中，可以兼具两者优势。

框架基于一个高效的**物体中心 3D 表示**，通过三阶段课程训练。

### 物体中心 3D 表示

- 使用 Mask3D 对输入 3D 场景进行实例分割
- 对每个物体实例提取：3D 视觉-几何特征（Uni3D）、2D 外观特征（DINOv2）
- 空间信息（物体位置和尺寸）通过可学习的位置编码（基于网格的高斯插值）编码
- 最终物体表示 = 视觉特征 + 空间嵌入的拼接，与指令 token 交错作为 LLM 输入
- 整个场景仅需约 400 个 token（对比视频方法的 10K-40K），非常高效

空间编码的具体公式：对每个空间轴 $a \in \{x, y, z, h, w, l\}$，定义可学习嵌入矩阵 $\mathbf{H}_a \in \mathbb{R}^{N \times d}$，其中 $N = 200$ 个均匀网格点。给定归一化配置 $p \in [0,1]^6$：

$$h_{\text{pos}}(p) = \sum_{a \in \{x,y,z,h,w,l\}} \sum_{k=1}^{N} w_k^a \cdot \mathbf{H}_k^a$$

其中高斯权重 $w_k^a = \exp\left(-(N \cdot p_a - k)^2\right)$，对越接近实际坐标的网格点赋予越高的权重。

### 阶段 1：感知对齐

**目标**：教模型"看"——将 3D 物体表示与 LLM 的文本嵌入空间对齐。

- 训练任务混合：物体识别（类别/属性）、定位（预测坐标）、描述（生成描述）
- 数据来源：ScanNet、MMScan 的标注
- 约 193K 指令-响应对
- 3 个 epoch，学习率 $1 \times 10^{-5}$，batch size 16

### 阶段 2：符号推理注入（CoT-SFT）

**目标**：教模型"逐步思考"——从符号程序中蒸馏推理模式。

**核心创新**：利用合成数据集的天然完整中间监督。

**程序原语**（沿用 NS3D 框架）：
- `scene()`：返回场景中所有物体集合 $O$
- `filter(O, condition)`：从物体集合中筛选满足语义条件的子集
- `relate(O_A, O_B, r)`：返回 $O_A$ 中与 $O_B$ 存在空间关系 $r$ 的物体
- `relate_triple`：三元组关系

**程序构建与数据**：
- Level 1（单步程序）：从 ScanNet 类别标注和 MMScan 属性标注构建约 78K 条 `filter` 操作轨迹
- Level 2（两步程序）：利用 Sr3D 数据集构建约 66K 条 `relate(filter(A), filter(B), r)` 轨迹
- 共计 144K 条带完整验证的 CoT 样本

**程序到 CoT 的转换**：
1. 解析程序的抽象语法树为执行序列 $S = \{s_1, \ldots, s_n\}$
2. 每步 $s_i$ 序列化为两个组件：
   - **Plan**：描述推理目标（如"找到所有类别为'花瓶'的物体"）
   - **Execution**：展示输入输出的物体详情（ID、位置、尺寸）
3. 最终 CoT = 所有 Plans 拼接 + 所有 Executions 拼接 + 摘要总结

**关键特点**：CoT 是空间锚定的——每个物体通过唯一 ID 和显式位置引用，可消歧同类别物体并进行精确空间验证。

**训练目标**（标准语言建模）：

$$\mathcal{L}_{\text{CoT-SFT}} = -\mathbb{E}_{(q, \text{CoT}, A) \sim \mathcal{D}} \left[ \log p_\theta(\text{CoT}, A \mid q, O) \right]$$

其中 $q$ 是任务指令，$O$ 是场景物体特征。1 个 epoch，学习率 $1 \times 10^{-5}$，batch size 8。

### 阶段 3：开放集和复杂推理泛化（CoT-RL）

**目标**：教模型适应真实世界的开放词汇指令和复杂嵌套查询。

**为什么需要 RL**：真实数据集（如 ScanRefer、Multi3DRefer）没有中间步骤的监督标注，只有最终答案是否正确。Stage 2 的 CoT-SFT 只能覆盖 Level 1-2 的程序结构，无法处理更深层嵌套和开放词汇。

**GRPO 优化**：采用 Group Relative Policy Optimization，对每个指令 $q$ 采样 $N$ 个响应，计算组归一化优势值。

**复合奖励函数**：

1. **Soft Grounding Reward**（解决 IoU 稀疏性问题）：

$$R_{\text{grounding}} = e^{-\alpha \|x_{\text{pred}} - x_{\text{gt}}\|_2^2} + e^{-\alpha \left\|\frac{s_{\text{pred}} - s_{\text{gt}}}{s_{\text{gt}}}\right\|_1}$$

其中 $x$ 是物体中心坐标，$s$ 是物体尺寸，$\alpha = 2$ 控制衰减率。这提供了基于空间接近度的密集反馈，即使初始预测零重叠也能有效引导探索。

2. **Format Reward**（防止推理结构退化）：

$$R_{\text{format}}(o) = \begin{cases} 1 & \text{if response contains valid plan and thinking tags, not too short} \\ 0 & \text{otherwise} \end{cases}$$

总奖励 $r_i = R_{\text{format}}(o_i) + R_{\text{grounding}}(o_i)$。

**训练配置**：GRPO，$G = 8$ 个 rollout/prompt，$\varepsilon = 0.2$，$\beta = 0$（无 KL 散度约束），学习率 $3 \times 10^{-6}$，batch size 128 prompts（1024 rollouts）。8 张 A100-40G GPU。

### 模块化推理增强

APEIRIA 独特的优势在于规划和感知的显式解耦，支持推理时的即插即用升级：

1. **外部规划注入**：可以用更强 LLM（如 Claude 4.5 Opus）生成的分解计划替换模型自生成的计划，模型作为视觉执行引擎。
2. **感知模块替换**：可以用更强的 3D 感知模型（如 SegDINO3D）替换 `scene()` 原语的输出，模型继续正常解码后续阶段。

两者都不需要重新训练。

## 实验与可视化

### 实验设置

**数据集与指标**：
- ScanRefer：3D 物体定位，Acc@0.25 / Acc@0.5
- Multi3DRefer：多物体定位，F1@0.25 / F1@0.5
- SQA3D：情境问答，Exact Match
- Scan2Cap：密集描述，CiDEr@IoU
- Nr3D：自然语言指令定位（零样本泛化测试）

**基线**：
- NS 方法：NS3D、LARC、LaSP
- 专家方法：ScanRefer、M3DRef-CLIP、3D-VisTA、SceneVerse
- 3D MLLMs：Grounded 3D-LLM、LLaVA-3D、PQ3D、Video-3D LLM、3D-LLaVA、Chat-Scene、Inst3D-LMM

**实现细节**：基于 Qwen3-VL-8B，LoRA ($r = 128$)，AdamW + Muon 优化器。

### 主要结果

**3D 空间推理（Table 1）**：
- ScanRefer Acc@0.25：APEIRIA 58.4，模块增强后 60.5（优于所有基线）
- Multi3DRefer F1@0.25：APEIRIA 59.2，模块增强后 60.9（优于所有基线）
- 相比最强 3D MLLM 基线（Inst3D-LMM: 57.8 / 58.3），分别提升 +0.6 / +0.9

**开放集概念泛化（Table 2）**：
- 仅在 Sr3D（合成数据）上训练，零样本在 Nr3D 上测试：36.5% 准确率
- 超越在 Nr3D 上完全监督训练的 NS3D（33.9%），提升 +2.6%
- 验证了推理模式可以泛化到训练中未见的开放词汇概念

**多任务泛化（Table 3）**：
- Scan2Cap：CiDEr@0.5 达 84.1（超越所有基线，包括 Video-3D LLM 的 83.8）
- SQA3D EM：58.6（匹配 Video-3D LLM）
- 无需修改阶段 1-2 的符号推理流程，只需更换阶段 3 的结果奖励

### 消融实验（Table 4）

- **去掉 Stage 3（无 CoT-RL）**：ScanRefer 下降 6.9%，Multi3DRefer 下降 3.9%——Stage 2 教会了推理"语法"，但 RL 对适应真实世界指令至关重要
- **去掉 Stage 2（无 CoT-SFT，直接从对齐到 RL）**：灾难性下降 10.2% / 22.5%——没有符号程序提供的结构化推理初始化，RL 探索空间从零开始极其困难
- **去掉 Format Reward**：下降 2.7% / 2.1%，出现"结构崩溃"——模型倾向于跳过推理直接回答
- **用 IoU 替换 Soft Grounding Reward**：轻微下降 0.7% / 0.5%
- **去掉 Thinking（直接输出答案）**：下降 1.6% / 1.0%——但仍然大幅优于 Stage 2，说明 RL 改善了内部表示

### 推理复杂度分析（Table 5）

- 短推理链（$\leq$4 步）：CoT-RL 微降 1.8%（SFT 已足够，RL 探索引入噪声）
- 中等推理链（5 步）：CoT-RL 提升 1.5%
- 长推理链（$\geq$6 步）：CoT-RL 提升 2.7%——验证了 CoT-RL 的核心价值：为 CoT-SFT 无法覆盖的复杂推理链提供监督信号

### 模块化分析（Table 6）

- **规划替换**（用 Claude 4.5 Opus 替换自生成计划）：仅微小提升 +0.2% / +0.3%——说明 APEIRIA 的蒸馏规划器已经非常稳健
- **感知替换**（用 SegDINO3D 替换）：显著提升 +2.0% / +1.4%，接近 Oracle 上界——说明感知精度是当前系统的主要瓶颈，但模块化架构使得无需重新训练即可升级

### 推理轨迹质量分析

使用 Claude Opus 4.6 作为自动评判器：
- RL 后 92.5% 的轨迹仍可解析为有效符号程序——RL 没有破坏阶段 2 注入的结构
- 正确样本比错误样本高 13.4% 的联合合规性——推理质量直接关联定位准确性

### 涌现行为

模型在 CoT-RL 阶段自发涌现了新的推理模式：
- 将 `filter` 原语扩展到开放词汇（如 `filter(beige chair)`）
- 自发组合出 `intersection` 和 `union` 等新的逻辑原语（课程中从未显式教授）来处理多条件约束

## 相关工作

**NS3D 方法**（NS3D, LARC, LaSP）：通过组合式程序实现可解释推理，但受限于封闭集词汇和需要密集逐步监督。APEIRIA 通过将推理模式蒸馏到 MLLM 中解决了这一限制，是首个在 NS3D 方法中引入基于结果的 RL。

**3D MLLMs**（Chat-Scene, LLaVA-3D, Inst3D-LMM 等）：分为物体中心方法（<1K token）和视频中心方法（10K-40K token），但都缺乏显式推理验证步骤。APEIRIA 采用物体中心架构以提高效率，同时注入了结构化的可解释推理轨迹。

**RL for Visual Reasoning**（3D-R1, Scene-R1）：并发工作尝试将 RL 应用于 3D 推理，但 3D-R1 的 CoT 由 LLM prompting 生成（易产生幻觉，缺乏显式空间锚定），Scene-R1 缺乏推理轨迹强制或初始化。APEIRIA 先从符号程序蒸馏验证过的空间锚定轨迹，再用 RL 精炼，提供了更稳健和可解释的空间推理。

## 潜在未来工作

1. **扩展到物理推理**：当前原语集针对静态空间几何，可扩展为包含物理感知谓词（如稳定性、支撑关系），处理"如果移走这个架子会怎样"类查询。
2. **室外和导航场景**：当前实验局限于室内 ScanNet 系列，可尝试加入领域特定谓词（如可通行性、智能体动态）扩展到室外场景。
3. **更深推理链的可靠性**：5 层以上嵌套操作的推理链可靠性和错误累积问题尚需进一步验证。
4. **更强感知模块集成**：随着 3D 感知技术进步（如更强的实例分割模型），可直接替换 `scene()` 模块获得性能提升，验证了框架的可持续升级性。
5. **降低感知瓶颈**：当前系统性能主要受限于 3D 感知精度，探索端到端感知-推理联合优化或更强的 3D 表示学习可能是方向。
